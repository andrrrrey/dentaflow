# Инструкция для роли «Администратор» — DentaFlow

> **Роль «Администратор»** — операционный сотрудник клиники (ресепшн, кол-центр). Управляет расписанием, пациентами, воронкой и коммуникациями. Финансовые отчёты, настройки системы и управление персоналом **недоступны**.

---

## Навигация

Боковое меню (десктоп) состоит из трёх блоков:

**Основное:**
- Обзор
- Расписание
- Пациенты
- Контроль звонков
- Контроль скриптов

**Коммуникации:**
- Коммуникации
- Воронка
- Задачи

**Прочее:**
- Маркетинг
- Справочники

Пункты **Аналитика**, **Отчёты**, **Сотрудники**, **Настройки** в меню скрыты — для роли «Администратор» они недоступны.

На мобильных устройствах внизу экрана расположена фиксированная панель навигации с 5–6 ярлыками и кнопкой «Ещё».

---

## 1. Обзор (`/`)

**Что отображается:**

В верхней части экрана — переключатель периода: **День / Неделя / Месяц**. Справа от него — человекочитаемые даты выбранного периода.

Ниже — **баннер ИИ-помощника**: текст с автоматическими советами по данным клиники. Кнопка **«Обновить»** запрашивает свежий анализ у ИИ.

Далее идут **KPI-карточки** (обновляются при смене периода):
- Новые лиды
- Записей создано
- Записей подтверждено
- Неявок
- Выручка
- Конверсия

Ниже в двух колонках:
- **Воронка пациентов** — столбиковая диаграмма по этапам (количество и %)
- **Источники** — таблица: канал, лидов, конверсия %

В нижнем ряду в двух колонках:
- **Нагрузка врачей** — список: имя врача, число приёмов, % загрузки
- **Рейтинг администраторов** — таблица с колонками: Имя, Конверсия %, Звонки, Оценка. Оценка подсвечивается цветом: зелёный ≥ 4,5, жёлтый ≥ 4,0, красный < 4,0

**Действия:**
| Элемент | Действие |
|---|---|
| Переключатель День/Неделя/Месяц | Меняет период для всех KPI и графиков |
| Кнопка «Обновить» в баннере ИИ | Запрашивает новый AI-анализ по текущему периоду |

## 2. Расписание (`/schedule`)

**Что отображается:**

Если на выбранный день есть записи — вверху автоматически появляется **баннер ИИ-ассистента «Расписание»**: показывает неподтверждённых пациентов, неявки, отмены, перегрузку врачей, пиковые часы и свободные слоты.

Под баннером — 4 **статистических карточки**:
- Всего записей
- Подтверждено (+ % выполнения)
- Отменено
- Выручка (сумма за день)

**Основной блок:** левая боковая панель + сетка расписания.

**Левая панель:**
- Мини-календарь: клик по дате выбирает день, стрелки листают месяц, кнопка «Сегодня» возвращает к текущей дате
- Кнопка **«Добавить запись»** — открывает модальное окно создания приёма
- Кнопка **«Синхронизировать»** — запускает синхронизацию расписания из 1Denta (кнопка становится серой с анимацией на время запроса)
- Фильтр **«Врач»** — выпадающий список всех врачей (выбор одного или «Все врачи»)
- Фильтр **«Статус»** — выпадающий список статусов: Подтверждено, Не подтверждено, Пришёл, Завершено, Отменено, Не явился
- Счётчик **«Визиты: N»**

**Сетка расписания:**
- Заголовок: дата и день недели, стрелки влево/вправо для перехода на соседний день
- Колонки врачей с числом записей
- Временная шкала с 09:00 до 20:00; блоки приёмов отрисованы пропорционально длительности
- Цвет блока зависит от статуса: зелёный — подтверждён, синий — не подтверждён, фиолетовый — пришёл/завершён, красный — отменён, жёлтый — не явился

**Модальное окно «Добавить запись»:** форма создания приёма (пациент, врач, дата, время, услуга, статус).

**Модальное окно «Карточка приёма»:** открывается кликом по блоку записи; показывает полную информацию о визите.

**Действия:**
| Элемент | Действие |
|---|---|
| Стрелки в мини-календаре | Листание по месяцам |
| Клик по дате | Переход на этот день в сетке |
| Кнопка «Сегодня» | Возврат к текущей дате |
| «Добавить запись» | Открывает форму создания приёма |
| «Синхронизировать» | Загружает расписание из 1Denta |
| Фильтры Врач / Статус | Фильтруют отображаемые блоки в сетке |
| Стрелки в заголовке сетки | Переход на предыдущий / следующий день |
| Клик на блок записи | Открывает детальную карточку приёма |

## 3. Пациенты (`/patients`)

**Что отображается:**

Страница-список всех пациентов базы данных.

Вверху — строка поиска с счётчиком «N пациентов». Поиск работает по имени, телефону, email. Рядом:
- Кнопка **«Фильтр»** — открывает/скрывает расширенную панель фильтров. Если применены фильтры, показывает их количество: «Фильтр (2)»
- Кнопка **«Синхр.»** — запускает синхронизацию данных из 1Denta; во время запроса показывает «Запрос...», после — «✓»

**Панель фильтров** (раскрывается кнопкой «Фильтр»):

Строка 1 — выпадающие списки:
- Пол: Все / Мужской / Женский
- Источник: Все / Telegram / Сайт / Звонок / Max/VK / Реферал
- Тип пациента: Все / Новые / Постоянные / Отказавшиеся / Потенциальные / Остальные
- Визиты: Все / Посетили / Не посетили

Строка 2 — диапазоны чисел:
- Сумма продаж, ₽: от — до
- Кол-во визитов: от — до

Строка 3 — диапазоны дат:
- День рождения: от — до
- Последнее посещение: от — до
- Добавлен в систему: от — до

Кнопки в панели:
- **«Применить»** — применяет выбранные фильтры
- **«Сбросить»** — очищает все фильтры

**Таблица пациентов** (колонки): Пациент (аватар + имя + email), Телефон, Источник, Последний визит, LTV (цветная оценка: зелёный ≥ 70, жёлтый ≥ 40, красный < 40), Выручка.

Клик по строке — открывает карточку пациента.

Внизу — **пагинация** (по 20 записей): номер страницы, стрелки назад/вперёд, номера страниц.

**Действия:**
| Элемент | Действие |
|---|---|
| Строка поиска | Мгновенная фильтрация по имени/телефону/email |
| «Фильтр» | Открывает/скрывает панель расширенных фильтров |
| «Применить» | Применяет фильтры, возвращает на первую страницу |
| «Сбросить» | Убирает все фильтры |
| «Синхр.» | Синхронизирует пациентов из 1Denta |
| Клик по строке пациента | Открывает карточку пациента |
| Пагинация | Навигация по страницам списка |

## 4. Карточка пациента (`/patients/:id`)

**Что отображается:**

Кнопка **«← Пациенты»** в левом верхнем углу — возврат к списку.

**Шапка пациента** — крупный блок с основными данными: имя, телефон, email, источник, дата рождения, статус (новый/постоянный). Две кнопки в шапке:
- **«Добавить сделку»** — открывает модальное окно создания сделки в воронке CRM
- **«Добавить запись»** — открывает модальное окно создания приёма в расписании

**Основная зона** делится на две части: слева — вкладки, справа — **блок ИИ-анализа** пациента.

**Вкладки:**

1. **История визитов** — список приёмов из 1Denta: дата, врач, услуга, статус, стоимость
2. **Статистика** — сводные показатели: число визитов, суммарная выручка, средний чек, LTV-балл, дата первого и последнего визита
3. **Коммуникации** — история входящих/исходящих сообщений и звонков по данному пациенту
4. **CRM** — сделки пациента в воронке с этапами и суммами
5. **Задачи** — список задач, связанных с этим пациентом

**Правая панель ИИ-анализа:** автоматически сформированный портрет пациента — поведенческие паттерны, риски отказа, рекомендации по работе.

**Действия:**
| Элемент | Действие |
|---|---|
| «← Пациенты» | Возврат к списку пациентов |
| «Добавить сделку» | Создание новой сделки по пациенту в воронке |
| «Добавить запись» | Создание приёма по пациенту в расписании |
| Вкладки | Переключение между блоками информации |

## 5. Контроль звонков (`/calls`)

**Что отображается:**

4 **статистических карточки** по выбранному периоду:
- Всего звонков
- Отвечено
- Пропущено
- % ответов

Ниже — блок «**История звонков**» с двумя фильтрами в шапке:
- **Период** (выпадающий список): 3 дня / 7 дней / 14 дней / 30 дней
- **Статус** (выпадающий список): Все / Отвечено / Пропущено

**Таблица звонков** (колонки): Дата+Время, Направление (иконка: входящий — синий, исходящий — фиолетовый), Откуда (номер), Куда (номер), Длительность (в формате М:СС), Статус (цветная метка), кнопка воспроизведения записи (если запись доступна).

Статус звонка: зелёный «Отвечен», красный «Пропущен», жёлтый (длительность < 30 сек).

**Действия:**
| Элемент | Действие |
|---|---|
| Фильтр «Период» | Меняет глубину истории и пересчитывает статистику |
| Фильтр «Статус» | Показывает только отвеченные или пропущенные |
| Иконка Play в строке | Открывает запись разговора в браузере |

## 6. Контроль скриптов (`/scripts`)

**Что отображается:**

Страница для загрузки, анализа и сравнения скриптов разговоров.

Вверху — кнопка **«Загрузить скрипт»**, которая открывает модальное окно.

**Модальное окно «Загрузить скрипт»:**
- Поле «Название» (обязательное)
- Поле «Категория» (необязательное, например «Входящий звонок»)
- Большое текстовое поле «Текст скрипта» (обязательное)
- Кнопка «Сохранить скрипт» / «Отмена»

**Список скриптов:** карточки с названием, категорией, датой создания. Для каждого скрипта:
- Кнопка **«Анализировать»** — ИИ анализирует скрипт и возвращает рекомендации: соответствие % , найденные проблемы, сильные стороны
- Кнопка **«Сравнить со звонком»** — позволяет выбрать звонок и сравнить его запись со скриптом; ИИ возвращает % соответствия и расхождения
- Кнопка **«Удалить»** — удаляет скрипт

**Блок результата анализа** (появляется после кнопки «Анализировать»):
- Общий % соответствия (зелёный ≥ 80%, жёлтый ≥ 60%, красный < 60%)
- Список выявленных проблем
- Список сильных сторон

**Действия:**
| Элемент | Действие |
|---|---|
| «Загрузить скрипт» | Открывает форму создания нового скрипта |
| «Анализировать» | Запускает ИИ-анализ скрипта |
| «Сравнить со звонком» | Сравнивает скрипт с выбранной записью звонка |
| «Удалить» | Удаляет скрипт из системы |

## 7. Коммуникации (`/communications`)

**Что отображается:**

Единый центр входящих обращений из всех каналов (Telegram, Телефония, Max/VK, Сайт).

Вверху — **строка фильтров** с тремя счётчиками-переключателями:
- **Все** (общий счётчик)
- **Новые** (синяя точка)
- **В работе** (жёлтая точка)
- **Готово** (зелёная точка)

Дополнительные фильтры: канал, приоритет.

**Основной блок** делится на две панели:

**Левая — список заявок:**
Каждая строка показывает:
- Иконку канала и тип (пропущенный звонок — красная иконка)
- Имя пациента (или «Новый контакт»)
- Цветную точку статуса
- Кнопку удаления (корзина)
- Канал, направление (стрелка вниз = входящий, вверх = исходящий), приоритет (Срочный / Высокий / Обычный / Низкий), время создания
- Превью содержимого (для звонка — длительность, для сообщения — текст)

Клик по строке выделяет её и открывает детали справа.

**Правая — панель деталей:**

Если ни одна заявка не выбрана — заглушка с иконкой и текстом «Выберите заявку для просмотра анализа».

Если заявка выбрана:
- Шапка: иконка канала, имя пациента, приоритет, канал, дата/время, текст сообщения или информация о звонке
- Блок **«ИИ-анализ»**: автоматическое резюме обращения, рекомендуемое следующее действие (выделено синим блоком), теги (например: «запись», «перенос», «боль»)
- Кнопка **«Добавить лид в воронку CRM»** — создаёт сделку в воронке с заполненными данными. После нажатия меняется на «Добавлено в воронку ✓» и становится неактивной

**Действия:**
| Элемент | Действие |
|---|---|
| Переключатели статуса | Фильтруют список по статусу обращения |
| Фильтры канал/приоритет | Дополнительная фильтрация списка |
| Клик по строке заявки | Открывает детальный вид с ИИ-анализом |
| Кнопка корзины в строке | Удаляет обращение |
| «Добавить лид в воронку CRM» | Создаёт новую сделку в Воронке на основе этого обращения |

## 8. Воронка (`/pipeline`)

**Что отображается:**

Страница управления CRM-воронкой пациентов.

Вверху — ссылка **«Настроить этапы воронки»** / **«Скрыть настройку этапов»**.

**Панель настройки этапов** (раскрывается кнопкой):
- Список этапов с цветной левой полосой
- Иконка захвата (⠿) — перетащить этап для изменения порядка
- Двойной клик по названию — открывает поле редактирования; Enter — сохранить, Escape — отменить
- Кнопка ✓ (галочка) — подтвердить переименование

Ниже — **баннер ИИ-ассистента «Воронка»**: автоматические советы (застрявшие сделки, горячие лиды, отказы, необработанные заявки).

**Панель управления:**
- Кнопка **«Добавить сделку»** (фиолетовая, слева)
- Выпадающий список **«Все ответственные»** — фильтр по назначенному администратору (список формируется из активных пользователей с ролью admin/manager)
- В режиме таблицы — дополнительный выпадающий список **«Все этапы»**
- Справа — сумма воронки: «Воронка: N ₽»
- Переключатель вида: иконки **Канбан** (сетка) и **Таблица** (список)

**Канбан-вид:**
Колонки по этапам воронки (Лист ожидания, Новые, Контакт, Переговоры, Записан, Лечение, Закрыто ✓, Закрыто ✗). В шапке колонки — название этапа, счётчик сделок, сумма. Карточки сделок можно перетаскивать между колонками. Клик по карточке — открывает модальное окно сделки.

**Табличный вид:**
Строки со столбцами: Пациент/Сделка, Услуга, Врач, Этап (цветная метка), Сумма, кнопка удаления. Клик по строке — открывает модальное окно сделки. Пагинация по 20 записей.

**Модальное окно сделки:** детальная карточка с историей, данными пациента, этапом, суммой, ответственным администратором, комментариями.

**Модальное окно «Добавить сделку»:** форма с полями — пациент, название, этап, услуга, сумма, врач.

**Боковая панель пациентов:** открывается кликом на счётчик пациентов в колонке; показывает список пациентов на этом этапе с именем, телефоном и меткой «Новый».

**Действия:**
| Элемент | Действие |
|---|---|
| «Настроить этапы воронки» | Раскрывает панель управления этапами |
| Перетаскивание этапа | Изменяет порядок этапов воронки |
| Двойной клик по названию этапа | Переименование этапа |
| «Добавить сделку» | Открывает форму создания новой сделки |
| Фильтр «Ответственные» | Показывает сделки только выбранного администратора |
| Фильтр «Этапы» (таблица) | Фильтрует сделки по этапу |
| Переключатель Канбан/Таблица | Меняет отображение воронки |
| Перетаскивание карточки | Перемещает сделку в другой этап |
| Клик по карточке/строке | Открывает детали сделки |
| Иконка корзины | Удаляет сделку (двойное подтверждение) |

## 9. Задачи (`/tasks`)

**Что отображается:**

Страница управления задачами по пациентам.

Вверху — три кнопки-переключателя статуса:
- **Все**
- **Активные**
- **Выполненные**

Рядом — красный счётчик «⚠ N просрочено» (если есть просроченные задачи).

Выпадающий список **«Все ответственные»** — фильтрация по назначенному администратору.

Кнопка **«+ Новая задача»** — открывает форму создания.

**Форма создания задачи** (появляется под строкой фильтров):
- **Тип**: Перезвонить / Напоминание / Подтвердить визит / Другое
- **Пациент** (необязательно): поле поиска пациентов с автодополнением
- **Название задачи**: текстовое поле
- **Срок**: дата и время (обязательное)
- **Ответственный**: выпадающий список активных администраторов и менеджеров
- Кнопка **«Создать»** (неактивна, если не заполнены обязательные поля)
- Кнопка **✕** — закрыть форму без создания

**Таблица задач** (колонки):
- Чекбокс выполнения (круг → галочка при клике)
- Задача: название (зачёркнутое — если выполнена)
- Тип: иконка + название типа (синяя трубка = Перезвонить, жёлтая стрелка = Напоминание, зелёный календарь = Подтвердить визит)
- Пациент: кликабельная ссылка на карточку пациента
- Ответственный: имя
- Срок: дата и время (красный — если просрочено)
- Статус: метка «В работе» (жёлтый) / «Просрочено» (красный) / «Выполнено» (зелёный)
- Кнопка удаления (корзина)

Внизу таблицы — **пагинация** (по 15 задач): номер страницы, стрелки, номера страниц.

**Действия:**
| Элемент | Действие |
|---|---|
| Кнопки Все/Активные/Выполненные | Фильтруют таблицу по статусу |
| Фильтр «Ответственные» | Показывает задачи только выбранного сотрудника |
| «+ Новая задача» | Показывает/скрывает форму создания задачи |
| Чекбокс в строке | Отмечает задачу выполненной / снимает отметку |
| Клик по имени пациента | Переход в карточку пациента |
| Кнопка корзины | Удаляет задачу |
| Пагинация | Переключение страниц |

## 10. Справочники (`/directories`)

**Что отображается:**

Страница просмотра данных, синхронизируемых из системы 1Denta.

Вверху — три вкладки:
- **Услуги** (иконка стетоскопа)
- **Врачи / Ресурсы** (иконка людей)
- **Товары** (иконка коробки)

Справа от вкладок — дата и время последнего обновления («Обновлено: 5 мая 2026, 14:32»).

Кнопка **«Обновить из 1Denta»** — запускает синхронизацию всех справочников. Во время загрузки показывает «Загрузка из 1Denta...» с анимацией.

**Вкладка «Услуги»:** таблица услуг клиники — название, код, категория, стоимость.

**Вкладка «Врачи / Ресурсы»:** список врачей и кабинетов — имя/название, специализация, статус активности.

**Вкладка «Товары»:** список товаров/расходников — название, артикул, единица измерения.

Данные в справочниках **доступны только для просмотра** — редактирование производится в системе 1Denta, а синхронизация подтягивает актуальные данные сюда.

**Действия:**
| Элемент | Действие |
|---|---|
| Вкладки Услуги/Врачи/Товары | Переключение между разделами справочника |
| «Обновить из 1Denta» | Синхронизирует данные из CRM 1Denta |
