# Инструкция для роли «Администратор» — DentaFlow

> **Роль «Администратор»** — операционный сотрудник клиники (ресепшн, кол-центр). Управляет расписанием, пациентами, воронкой и коммуникациями. Финансовые отчёты, настройки системы и управление персоналом **недоступны**.

---

## Навигация

Боковое меню (десктоп) состоит из трёх блоков:

**Основное:**
- Обзор
- Расписание
- Пациенты
- Контроль звонков
- Контроль скриптов

**Коммуникации:**
- Каналы коммуникаций (Telegram, Max, Новофон)
- Коммуникации
- Воронка
- Задачи

**Прочее:**
- Маркетинг
- Справочники

Пункты **Аналитика**, **Отчёты**, **Сотрудники**, **Настройки** в меню скрыты — для роли «Администратор» они недоступны.

**Сворачивание меню.** В боковом меню есть кнопка сворачивания/разворачивания. В свёрнутом виде меню превращается в узкую полосу с иконками и всплывающими подсказками (tooltip) при наведении. Выбранное состояние запоминается и сохраняется между сессиями.

**Счётчик задач.** На пункте **«Задачи»** отображается красный бейдж с числом активных (невыполненных) задач. Бейдж обновляется автоматически при выполнении или удалении задач, а также раз в минуту. В свёрнутом меню счётчик показывается компактной точкой/числом на иконке.

На мобильных устройствах внизу экрана расположена фиксированная панель навигации с 5–6 ярлыками и кнопкой «Ещё». Счётчик активных задач дублируется и в мобильной навигации.

## 1. Обзор (`/`)

**Что отображается:**

В верхней части экрана — переключатель периода: **День / Неделя / Месяц**. Справа от него — человекочитаемые даты выбранного периода.

Ниже — **баннер ИИ-помощника**: текст с автоматическими советами по данным клиники. Кнопка **«Обновить»** запрашивает свежий анализ у ИИ.

Далее идут **KPI-карточки** (обновляются при смене периода):
- Новые лиды
- Записей создано
- Записей подтверждено
- Неявок
- Выручка
- Конверсия

Ниже в двух колонках:
- **Воронка пациентов** — столбиковая диаграмма по этапам (количество и %)
- **Источники** — таблица: канал, лидов, конверсия %

В нижнем ряду в двух колонках:
- **Нагрузка врачей** — список: имя врача, число приёмов, % загрузки
- **Рейтинг администраторов** — таблица системы мотивации с колонками: **Место** (для топ-3 — медали 🥇/🥈/🥉, далее номер), **Администратор** (аватар + имя), **Выполнено задач**, **Баллы** (с иконкой звезды). Список отсортирован по сумме набранных баллов. Баллы начисляются автоматически за выполненные действия (в первую очередь — за выполнение задач). Если баллы ещё не начислялись, показывается сообщение «Баллы ещё не начислялись. Рейтинг появится после первых выполненных задач.»

**Действия:**
| Элемент | Действие |
|---|---|
| Переключатель День/Неделя/Месяц | Меняет период для всех KPI и графиков |
| Кнопка «Обновить» в баннере ИИ | Запрашивает новый AI-анализ по текущему периоду |

## 2. Расписание (`/schedule`)

**Что отображается:**

> **Изменение:** верхний баннер ИИ-ассистента «Расписание» убран — страница начинается сразу со статистических карточек.

Вверху — 4 **статистических карточки**:
- Всего записей
- Подтверждено (+ % выполнения)
- Отменено
- Выручка (сумма за день)

**Основной блок:** левая боковая панель + сетка расписания.

**Левая панель:**
- Мини-календарь: клик по дате выбирает день, стрелки листают месяц, кнопка «Сегодня» возвращает к текущей дате
- Кнопка **«Добавить запись»** — открывает модальное окно создания приёма
- Кнопка **«Синхронизировать»** — запускает синхронизацию расписания из 1Denta (кнопка становится серой с анимацией на время запроса)
- Фильтр **«Врач»** — выпадающий список всех врачей (выбор одного или «Все врачи»)
- Фильтр **«Статус»** — выпадающий список статусов: Подтверждено, Не подтверждено, Пришёл, Завершено, Отменено, Не явился
- Счётчик **«Визиты: N»**

**Сетка расписания:**
- Заголовок: дата и день недели, стрелки влево/вправо для перехода на соседний день
- **Шапка врачей** — для каждого врача показывается аватар, имя и специализация/должность (берётся из справочника ресурсов 1Denta), а также число записей. Колонки врачей сделаны шире для читаемости.
- Временная шкала с 09:00 до 20:00; блоки приёмов отрисованы пропорционально длительности. Слоты увеличены по высоте, карточки приёмов крупнее — прямо на карточке видны имя пациента, телефон и услуга.
- Цвет блока зависит от статуса: зелёный — подтверждён, синий — не подтверждён, фиолетовый — пришёл/завершён, красный — отменён, жёлтый — не явился
- **Закрепление при прокрутке** — сетка имеет собственную область прокрутки по обеим осям: строка с именами врачей закреплена сверху при вертикальной прокрутке, а колонка времени — слева при горизонтальной. Левый верхний угол остаётся зафиксированным в обоих направлениях.

**Модальное окно «Добавить запись»:** форма создания приёма (пациент, врач, дата, время, услуга, статус).

**Модальное окно «Карточка приёма»:** открывается кликом по блоку записи; показывает полную информацию о визите.

**Действия:**
| Элемент | Действие |
|---|---|
| Стрелки в мини-календаре | Листание по месяцам |
| Клик по дате | Переход на этот день в сетке |
| Кнопка «Сегодня» | Возврат к текущей дате |
| «Добавить запись» | Открывает форму создания приёма |
| «Синхронизировать» | Загружает расписание из 1Denta |
| Фильтры Врач / Статус | Фильтруют отображаемые блоки в сетке |
| Стрелки в заголовке сетки | Переход на предыдущий / следующий день |
| Клик на блок записи | Открывает детальную карточку приёма |

## 3. Пациенты (`/patients`)

**Что отображается:**

Страница-список всех пациентов базы данных.

Вверху — строка поиска с счётчиком «N пациентов». Поиск работает по имени, телефону, email. Рядом:
- Кнопка **«Фильтр»** — открывает/скрывает расширенную панель фильтров. Если применены фильтры, показывает их количество: «Фильтр (2)»
- Кнопка **«Синхр.»** — запускает синхронизацию данных из 1Denta; во время запроса показывает «Запрос...», после — «✓»

**Панель фильтров** (раскрывается кнопкой «Фильтр»):

Строка 1 — выпадающие списки:
- Пол: Все / Мужской / Женский
- Источник: Все / Telegram / Сайт / Звонок / Max/VK / Реферал
- Тип пациента: Все / Новые / Постоянные / Отказавшиеся / Потенциальные / Остальные
- Визиты: Все / Посетили / Не посетили

Строка 2 — диапазоны чисел:
- Сумма продаж, ₽: от — до
- Кол-во визитов: от — до

Строка 3 — диапазоны дат:
- День рождения: от — до
- Последнее посещение: от — до
- Добавлен в систему: от — до

Кнопки в панели:
- **«Применить»** — применяет выбранные фильтры
- **«Сбросить»** — очищает все фильтры

**Таблица пациентов** (колонки): Пациент (аватар + имя + email), Телефон, Источник, Последний визит, LTV (цветная оценка: зелёный ≥ 70, жёлтый ≥ 40, красный < 40), Выручка.

Клик по строке — открывает карточку пациента.

Внизу — **пагинация** (по 20 записей): номер страницы, стрелки назад/вперёд, номера страниц.

**Действия:**
| Элемент | Действие |
|---|---|
| Строка поиска | Мгновенная фильтрация по имени/телефону/email |
| «Фильтр» | Открывает/скрывает панель расширенных фильтров |
| «Применить» | Применяет фильтры, возвращает на первую страницу |
| «Сбросить» | Убирает все фильтры |
| «Синхр.» | Синхронизирует пациентов из 1Denta |
| Клик по строке пациента | Открывает карточку пациента |
| Пагинация | Навигация по страницам списка |

## 4. Карточка пациента (`/patients/:id`)

**Что отображается:**

Кнопка **«← Пациенты»** в левом верхнем углу — возврат к списку.

**Шапка пациента** — крупный блок с основными данными: имя, телефон, email, источник, дата рождения, статус (новый/постоянный). Две кнопки в шапке:
- **«Добавить сделку»** — открывает модальное окно создания сделки в воронке CRM
- **«Добавить запись»** — открывает модальное окно создания приёма в расписании

**Основная зона** делится на две части: слева — вкладки, справа — **блок ИИ-анализа** пациента.

**Вкладки:**

1. **История визитов** — список приёмов из 1Denta: дата, врач, услуга, статус, стоимость
2. **Статистика** — сводные показатели: число визитов, суммарная выручка, средний чек, LTV-балл, дата первого и последнего визита
3. **Коммуникации** — история входящих/исходящих сообщений и звонков по данному пациенту
4. **CRM** — сделки пациента в воронке с этапами и суммами
5. **Задачи** — список задач, связанных с этим пациентом

**Правая панель ИИ-анализа:** автоматически сформированный портрет пациента — поведенческие паттерны, риски отказа, рекомендации по работе. Блок **«ИИ-Анализ пациента»** запускается автоматически при открытии карточки. Результат кэшируется на стороне сервера (по «отпечатку» данных пациента): пока данные не изменились с прошлого открытия, используется готовый анализ, а не генерируется заново. Кнопка **«Обновить»** принудительно запрашивает свежий анализ.

**Действия:**
| Элемент | Действие |
|---|---|
| «← Пациенты» | Возврат к списку пациентов |
| «Добавить сделку» | Создание новой сделки по пациенту в воронке |
| «Добавить запись» | Создание приёма по пациенту в расписании |
| Вкладки | Переключение между блоками информации |

## 5. Контроль звонков (`/calls`)

> Раздел работает через интеграцию с IP-телефонией **Новофон**. Данные о входящих и исходящих звонках поступают автоматически через вебхук. Пропущенные звонки автоматически создают задачу «Перезвонить» с высоким приоритетом в разделе «Задачи».
>
> **Дедупликация звонков.** Новофон отправляет несколько вебхуков на один сеанс звонка (входящий / завершён / пропущен). Система объединяет их в одну строку истории, чтобы «Контроль звонков» совпадал с отчётом Новофон. Звонок считается отвеченным, только если линия была действительно поднята. Задачи «Перезвонить» по пропущенным создаются без дублей.
>
> **Сверка статусов при синхронизации.** Статус звонка (отвечен/пропущен), длительность, направление и время определяются по итоговому исходу АТС (disposition) Новофон. При синхронизации система перезаписывает сохранённые строки авторитетными данными Новофон — например, короткие звонки на «1 сек» будут показаны корректно после синхронизации.

**Что отображается:**

4 **статистических карточки** по выбранному периоду:
- Всего звонков
- Отвечено
- Пропущено
- % ответов

Ниже — блок «**История звонков**» с двумя фильтрами в шапке:
- **Период** (выпадающий список): 3 дня / 7 дней / 14 дней / 30 дней
- **Статус** (выпадающий список): Все / Отвечено / Пропущено

**Таблица звонков** (колонки): Дата+Время, Направление (иконка: входящий — синий, исходящий — фиолетовый), Откуда (номер), Куда (номер), Длительность (в формате М:СС), Статус (цветная метка), кнопка воспроизведения записи (если запись доступна).

Статус звонка: зелёный «Отвечен», красный «Пропущен», жёлтый (длительность < 30 сек).

**Действия:**
| Элемент | Действие |
|---|---|
| Фильтр «Период» | Меняет глубину истории и пересчитывает статистику |
| Фильтр «Статус» | Показывает только отвеченные или пропущенные |
| Иконка Play в строке | Открывает запись разговора в браузере |

## 6. Контроль скриптов (`/scripts`)

**Что отображается:**

Страница для загрузки, анализа и сравнения скриптов разговоров.

Вверху — кнопка **«Загрузить скрипт»**, которая открывает модальное окно.

**Модальное окно «Загрузить скрипт»:**
- Поле «Название» (обязательное)
- Поле «Категория» (необязательное, например «Входящий звонок»)
- Большое текстовое поле «Текст скрипта» (обязательное)
- Кнопка «Сохранить скрипт» / «Отмена»

**Список скриптов:** карточки с названием, категорией, датой создания. Для каждого скрипта:
- Кнопка **«Анализировать»** — ИИ анализирует скрипт и возвращает рекомендации: соответствие % , найденные проблемы, сильные стороны
- Кнопка **«Сравнить со звонком»** — позволяет выбрать звонок и сравнить его запись со скриптом; ИИ возвращает % соответствия и расхождения
- Кнопка **«Удалить»** — удаляет скрипт

**Блок результата анализа** (появляется после кнопки «Анализировать»):
- Общий % соответствия (зелёный ≥ 80%, жёлтый ≥ 60%, красный < 60%)
- Список выявленных проблем
- Список сильных сторон

**Действия:**
| Элемент | Действие |
|---|---|
| «Загрузить скрипт» | Открывает форму создания нового скрипта |
| «Анализировать» | Запускает ИИ-анализ скрипта |
| «Сравнить со звонком» | Сравнивает скрипт с выбранной записью звонка |
| «Удалить» | Удаляет скрипт из системы |

## 7. Каналы коммуникаций

В DentaFlow подключены три канала приёма обращений от пациентов. Все входящие обращения из этих каналов автоматически появляются в разделе **«Коммуникации»**.

---

### 7.1 Telegram-бот

Telegram-бот — это AI-ассистент клиники, доступный пациентам в мессенджере Telegram.

**Навигация через командное меню:**

В нижней части окна чата с ботом есть иконка **≡** («Меню команд»). При нажатии открывается список доступных команд:

| Команда | Действие |
|---|---|
| `/start` или `/menu` | Главное меню — приветственное сообщение |
| `/book` | 📅 Записаться на приём |
| `/ask` | 💬 Задать вопрос (AI-чат) |
| `/manager` | 📞 Связаться с менеджером |

Эти команды всегда доступны через меню и не дублируются кнопками в сообщениях — интерфейс чата остаётся чистым.

**Что может делать пациент через бота:**

| Действие | Что происходит |
|---|---|
| 📅 Записаться на приём | Бот пошагово собирает имя, телефон и описание проблемы. На шаге ввода телефона появляется кнопка **«📱 Поделиться номером»** — пациент может передать номер одним касанием без ручного ввода. После заполнения — создаётся обращение в разделе «Коммуникации» |
| 💬 Задать вопрос | Открывается AI-чат. Пациент может задать любой вопрос об услугах, ценах, расписании. AI отвечает на основе базы знаний клиники |
| 📞 Связаться с менеджером | Бот собирает имя и телефон (с кнопкой «📱 Поделиться номером»), создаёт обращение с указанием имени и номера клиента |

**Двусторонняя переписка:**

После того как пациент оставил контакты через любой флоу, устанавливается режим открытого диалога:
- Все последующие сообщения пациента в Telegram автоматически поступают в раздел «Коммуникации» → вкладку «Переписка» нужного обращения
- Администратор может отвечать напрямую из DentaFlow — пациент получит сообщение в Telegram
- Пациент видит подтверждение каждого полученного сообщения: «✉️ Получено, менеджер ответит вам здесь.»

**Автоматические напоминания:** за 24 часа до записи пациент, который ранее взаимодействовал с ботом, получает сообщение-напоминание с датой, временем, именем врача и услугой. В напоминании есть кнопки «Перенести» и «Отказаться» — обе направляют пациента позвонить в клинику.

**Для администратора:** все заявки из Telegram отображаются в разделе «Коммуникации» с иконкой канала Telegram. Имя пациента отображается так, как он его ввёл в боте.

### 7.2 Max-бот (Max.ru / ВКонтакте)

Max-бот — аналог Telegram-бота, но работает в мессенджере Max (экосистема ВКонтакте / VK).

**Отличие от Telegram-бота:** Max не поддерживает командное меню (аналога кнопки ≡ нет). Поэтому три основных действия (📅 Записаться, 💬 Задать вопрос, 📞 Связаться с менеджером) отображаются в виде кнопок прямо в сообщении главного меню.

**Функционал:**
- Три инлайн-кнопки в главном меню: запись, AI-вопрос, связь с менеджером
- Сбор контактов при записи и запросе звонка
- AI-чат на основе знаний клиники
- Автоматические напоминания за 24 часа до визита
- **Двусторонняя переписка** — работает так же, как в Telegram: сообщения пациента попадают в «Переписку» в DentaFlow, администратор может отвечать из интерфейса

Обращения из Max попадают в раздел «Коммуникации» с пометкой канала **Max/VK**.

## 8. Коммуникации (`/communications`)

**Что отображается:**

Единый центр входящих обращений из всех каналов: **Telegram-бот**, **Max/VK-бот**, **Новофон (телефония)**, **Сайт**.

Все заявки от ботов, звонки и прочие обращения автоматически попадают сюда без участия администратора.

Вверху — **строка фильтров** с тремя счётчиками-переключателями:
- **Все** (общий счётчик)
- **Новые** (синяя точка)
- **В работе** (жёлтая точка)
- **Готово** (зелёная точка)

Дополнительные фильтры: канал, приоритет.

**Основной блок** делится на две панели:

**Левая — список заявок:**
Каждая строка показывает:
- Иконку канала и тип (пропущенный звонок — красная иконка)
- Имя пациента (или «Новый контакт»)
- Цветную точку статуса
- Кнопку удаления (корзина)
- Канал, направление (стрелка вниз = входящий, вверх = исходящий), приоритет (Срочный / Высокий / Обычный / Низкий), время создания
- Превью содержимого (для звонка — длительность, для сообщения — текст)

Клик по строке выделяет её и открывает детали справа.

**Правая — панель деталей:**

Если ни одна заявка не выбрана — заглушка с иконкой и текстом «Выберите заявку для просмотра анализа».

Если заявка выбрана:
- Шапка: иконка канала, имя пациента (отображается реальное имя, введённое клиентом в боте), приоритет, канал, дата/время, текст сообщения или информация о звонке
- Блок **«ИИ-анализ»**: автоматическое резюме обращения, рекомендуемое следующее действие (выделено синим блоком), теги (например: «запись», «перенос», «боль»)
- Кнопка **«Добавить лид в воронку CRM»** — создаёт сделку в воронке с заполненными данными. После нажатия меняется на «Добавлено в воронку ✓» и становится неактивной
- Блок **«Переписка»** (только для каналов Telegram и Max) — окно двустороннего чата с клиентом:
  - Сообщения клиента отображаются слева (синий фон), ответы администратора — справа (фиолетовый фон)
  - Поле ввода с кнопкой **«Отправить»**: нажмите **Enter** для отправки, **Shift+Enter** — чтобы перейти на новую строку
  - Если обращение создано до введения функции переписки — вместо поля ввода отображается информационное сообщение о необходимости связаться с клиентом напрямую по телефону

**Действия:**
| Элемент | Действие |
|---|---|
| Переключатели статуса | Фильтруют список по статусу обращения |
| Фильтры канал/приоритет | Дополнительная фильтрация списка |
| Клик по строке заявки | Открывает детальный вид с ИИ-анализом |
| Кнопка корзины в строке | Удаляет обращение |
| «Добавить лид в воронку CRM» | Создаёт новую сделку в Воронке на основе этого обращения |
| Поле «Переписка» + Enter | Отправляет ответ клиенту через Telegram или Max |

## 9. Воронка (`/pipeline`)

**Что отображается:**

Страница управления CRM-воронкой пациентов.

Вверху — ссылка **«Настроить этапы воронки»** / **«Скрыть настройку этапов»**.

**Панель настройки этапов** (раскрывается кнопкой):
- Список этапов с цветной левой полосой
- Иконка захвата (⠿) — перетащить этап для изменения порядка
- Двойной клик по названию — открывает поле редактирования; Enter — сохранить, Escape — отменить
- Кнопка ✓ (галочка) — подтвердить переименование

Ниже — **баннер ИИ-ассистента «Воронка»**: автоматические советы (застрявшие сделки, горячие лиды, отказы, необработанные заявки).

**Панель управления:**
- Кнопка **«Добавить сделку»** (фиолетовая, слева)
- Выпадающий список **«Все ответственные»** — фильтр по назначенному администратору (список формируется из активных пользователей с ролью admin/manager)
- В режиме таблицы — дополнительный выпадающий список **«Все этапы»**
- Справа — сумма воронки: «Воронка: N ₽»
- Переключатель вида: иконки **Канбан** (сетка) и **Таблица** (список)

**Канбан-вид:**
Колонки по этапам воронки (Лист ожидания, Новые, Контакт, Переговоры, Записан, Лечение, Закрыто ✓, Закрыто ✗). В шапке колонки — название этапа, счётчик сделок, сумма. Карточки сделок можно перетаскивать между колонками. Клик по карточке — открывает модальное окно сделки.

**Табличный вид:**
Строки со столбцами: Пациент/Сделка, Услуга, Врач, Этап (цветная метка), Сумма, кнопка удаления. Клик по строке — открывает модальное окно сделки. Пагинация по 20 записей.

**Модальное окно сделки:** детальная карточка с историей, данными пациента, этапом, суммой, ответственным администратором, комментариями.

**Модальное окно «Добавить сделку»:** форма с полями — пациент, название, этап, услуга, сумма, врач.

**Боковая панель пациентов:** открывается кликом на счётчик пациентов в колонке; показывает список пациентов на этом этапе с именем, телефоном и меткой «Новый».

**Действия:**
| Элемент | Действие |
|---|---|
| «Настроить этапы воронки» | Раскрывает панель управления этапами |
| Перетаскивание этапа | Изменяет порядок этапов воронки |
| Двойной клик по названию этапа | Переименование этапа |
| «Добавить сделку» | Открывает форму создания новой сделки |
| Фильтр «Ответственные» | Показывает сделки только выбранного администратора |
| Фильтр «Этапы» (таблица) | Фильтрует сделки по этапу |
| Переключатель Канбан/Таблица | Меняет отображение воронки |
| Перетаскивание карточки | Перемещает сделку в другой этап |
| Клик по карточке/строке | Открывает детали сделки |
| Иконка корзины | Удаляет сделку (двойное подтверждение) |

## 10. Задачи (`/tasks`)

**Что отображается:**

Страница управления задачами по пациентам.

Вверху — четыре кнопки-переключателя статуса:
- **Все**
- **Активные**
- **Выполненные**
- **Архив** — истёкшие автозадачи (невыполненные автоматические задачи прошлых дней, деактивированные системой). Если архив пуст, показывается «Архив пуст — истёкших задач нет».

Рядом — красный счётчик «⚠ N просрочено» (если есть просроченные задачи).

Выпадающий список **«Все ответственные»** — фильтрация по назначенному администратору.

Кнопка **«+ Новая задача»** — открывает форму создания.

**Форма создания задачи** (появляется под строкой фильтров):
- **Тип**: Перезвонить / Напоминание / Подтвердить визит / Другое
- **Пациент** (необязательно): поле поиска пациентов с автодополнением
- **Название задачи**: текстовое поле
- **Срок**: дата и время (обязательное)
- **Ответственный**: выпадающий список активных администраторов и менеджеров
- Кнопка **«Создать»** (неактивна, если не заполнены обязательные поля)
- Кнопка **✕** — закрыть форму без создания

**Таблица задач** (колонки):
- Чекбокс выполнения (круг → галочка при клике)
- Задача: название (зачёркнутое — если выполнена). У автоматически созданных задач рядом с названием — синий бейдж **⚡ Авто** («Автоматически создана системой»). У выполненной задачи под названием показывается дата и время выполнения.
- Тип: иконка + название типа (синяя трубка = Перезвонить, жёлтая стрелка = Напоминание, зелёный календарь = Подтвердить визит)
- Пациент: кликабельная ссылка на карточку пациента
- Ответственный: имя сотрудника, выполнившего задачу. Для невыполненной автозадачи без ответственного отображается «Общая задача».
- Срок: дата и время (красный — если просрочено)
- Статус: метка «В работе» (жёлтый) / «Просрочено» (красный) / «Выполнено» (зелёный)
- Кнопка удаления (корзина)

Внизу таблицы — **пагинация** (по 15 задач): номер страницы, стрелки, номера страниц.

**Автоматические задачи и система баллов:**

Система ежедневно сама создаёт задачи для администраторов (помечены бейджем **⚡ Авто**):

| Когда | Что создаётся |
|---|---|
| Утром (07:00 МСК) | По каждой записи на сегодня (со статусом «Запланирован»/«Подтверждён») создаётся задача типа «Подтвердить визит»: «Позвонить: {имя пациента} — {услуга} — {время}». Срок — до конца дня. |
| Утром (08:00 МСК) | По вчерашним визитам система запрашивает актуальные данные из 1Denta. Если пациент **пришёл** на приём (attendance), но в карточке визита нет записей о приёме (нет проданных услуг/суммы оплаты/комментария), создаётся задача: «Внести записи о приёме в МИС — {имя} — {услуга} ({время} вчера): указать проданные услуги». Если 1Denta не настроен — задача пропускается. Повторный запуск в тот же день дублей не создаёт. |
| Ночью (00:05 МСК) | Невыполненные автозадачи прошлых дней деактивируются и попадают во вкладку **«Архив»**. |

**Баллы за выполнение.** При отметке задачи выполненной сотруднику, который её закрыл, автоматически начисляются баллы (по умолчанию 10 за задачу). Накопленные баллы формируют **«Рейтинг администраторов»** на странице «Обзор». Размер баллов за разные действия настраивается в разделе «Настройки → Мотивация» (доступен ролям «Владелец»/«Менеджер»).

**Действия:**
| Элемент | Действие |
|---|---|
| Кнопки Все/Активные/Выполненные/Архив | Фильтруют таблицу по статусу (Архив — истёкшие автозадачи) |
| Фильтр «Ответственные» | Показывает задачи только выбранного сотрудника |
| «+ Новая задача» | Показывает/скрывает форму создания задачи |
| Чекбокс в строке | Отмечает задачу выполненной / снимает отметку |
| Клик по имени пациента | Переход в карточку пациента |
| Кнопка корзины | Удаляет задачу |
| Пагинация | Переключение страниц |

## 11. Справочники (`/directories`)

**Что отображается:**

Страница просмотра данных, синхронизируемых из системы 1Denta.

Вверху — три вкладки:
- **Услуги** (иконка стетоскопа)
- **Врачи / Ресурсы** (иконка людей)
- **Товары** (иконка коробки)

Справа от вкладок — дата и время последнего обновления («Обновлено: 5 мая 2026, 14:32»).

Кнопка **«Обновить из 1Denta»** — запускает синхронизацию всех справочников. Во время загрузки показывает «Загрузка из 1Denta...» с анимацией.

**Вкладка «Услуги»:** таблица услуг клиники — название, код, категория, стоимость.

**Вкладка «Врачи / Ресурсы»:** список врачей и кабинетов — имя/название, специализация, статус активности.

**Вкладка «Товары»:** список товаров/расходников — название, артикул, единица измерения.

Данные в справочниках **доступны только для просмотра** — редактирование производится в системе 1Denta, а синхронизация подтягивает актуальные данные сюда.

**Действия:**
| Элемент | Действие |
|---|---|
| Вкладки Услуги/Врачи/Товары | Переключение между разделами справочника |
| «Обновить из 1Denta» | Синхронизирует данные из CRM 1Denta |

## Последние обновления функционала

Этот раздел фиксирует изменения, добавленные в последних итерациях разработки.

---

### Расписание

- **Удаление записи** — в карточке приёма появилась кнопка удаления. Удаление требует подтверждения и автоматически синхронизируется с 1Denta.
- **Проверка конфликтов** — при создании или редактировании записи система предупреждает, если у выбранного врача уже есть приём в это время.
- **Выбор услуги при записи** — форма создания приёма теперь передаёт конкретные услуги в 1Denta. Если услуга не выбрана, запись создаётся без привязки к услуге.
- **Фильтр онлайн-записи** — при синхронизации расписания из 1Denta система учитывает только услуги с включённой онлайн-записью.
- **Индикатор синхронизации** — кнопка «Синхронизировать» показывает статус процесса во время загрузки данных из 1Denta.

---

### Пациенты

- **Новый источник «Max/VK»** — в фильтре «Источник» добавлен канал Max/VK. Пациенты, пришедшие через Max-бот, помечаются этим источником.

---

### Воронка (CRM)

- **Сохранение состояния кнопки «Добавить лид»** — если лид из обращения уже был добавлен в воронку, кнопка остаётся неактивной даже после перезагрузки страницы (состояние сохраняется в браузере).
- **Автозаполнение сделки** — при нажатии «Добавить лид в воронку CRM» система автоматически извлекает имя и телефон из содержимого обращения и заполняет поля новой сделки.
- **Отключена автосинхронизация «Листа ожидания»** — список пациентов в стадии «Лист ожидания» больше не запрашивается автоматически при каждом открытии страницы воронки.

### Боты и напоминания

- **Автоматические напоминания о записи** — за 24 часа до визита пациент получает сообщение через Telegram или Max (если он ранее запустил бота и предоставил номер телефона). В напоминании указаны услуга, дата, время и имя врача.
- **Токен бота из базы данных** — настройки Telegram-бота и Max-бота теперь хранятся в базе данных и управляются через раздел «Настройки» (доступен роли «Владелец»/«Менеджер»), а не задаются жёстко при деплое.
- **Командное меню Telegram** — основные действия бота (Записаться, Задать вопрос, Связаться с менеджером) вынесены в командное меню (кнопка ≡ в Telegram). Сообщения бота больше не содержат три инлайн-кнопки — интерфейс чата остаётся чистым.
- **Кнопка «Поделиться номером»** — на шаге ввода телефона в Telegram теперь появляется нативная кнопка «📱 Поделиться номером». Пациент передаёт свой номер одним касанием — без ручного ввода.
- **Двусторонняя переписка** — после того как клиент оставил контакты через бот, открывается режим диалога. Все последующие сообщения клиента в Telegram или Max автоматически поступают в раздел «Коммуникации» DentaFlow. Администратор может отвечать прямо из DentaFlow — ответ доставляется клиенту в мессенджер.
- **Отображение реального имени клиента** — в разделе «Коммуникации» для обращений из Telegram-бота и Max-бота теперь отображается имя, которое клиент ввёл в ходе диалога (раньше показывалось «Новый контакт»).

### Коммуникации

- **Блок «Переписка»** — в правой панели детального просмотра обращения из Telegram или Max появился чат. Входящие сообщения клиента отображаются слева, ответы администратора — справа. Поле ввода поддерживает Enter для отправки и Shift+Enter для переноса строки.
- **Реальное имя в списке заявок** — для обращений, созданных через Telegram-бот или Max-бот, в левом списке теперь отображается имя клиента вместо «Новый контакт».

### Обновления июнь 2026

**Интерфейс и навигация**
- **Сворачиваемое боковое меню** — добавлена кнопка сворачивания/разворачивания. В свёрнутом виде меню становится узкой полосой с иконками и всплывающими подсказками. Состояние запоминается между сессиями.
- **Счётчик активных задач** — на пункте «Задачи» (в боковом и мобильном меню) появился красный бейдж с числом невыполненных задач; обновляется при изменении задач и раз в минуту.

**Обзор и система мотивации администраторов**
- **Балльный рейтинг администраторов** — прежний блок «Рейтинг администраторов» (Конверсия %, Звонки, Оценка) заменён на единый рейтинг с баллами: Место (медали для топ-3), Администратор, Выполнено задач, Баллы. Тот же рейтинг используется на странице «Сотрудники».
- **Начисление баллов** — за выполнение задач (и других действий) администраторам автоматически начисляются баллы. Настройка стоимости действий — в «Настройки → Мотивация» (роль «Владелец»/«Менеджер»).

**Задачи и автоматизация**
- **Автозадачи обзвона на сегодня** — каждое утро (07:00 МСК) по записям текущего дня создаются задачи «Позвонить: {пациент} — {услуга} — {время}» (тип «Подтвердить визит»).
- **Follow-up по вчерашним визитам через 1Denta** — каждое утро (08:00 МСК) система сверяется с 1Denta: если пациент пришёл на приём, но записи о приёме в МИС нет (нет проданных услуг/оплаты/комментария), создаётся задача «Внести записи о приёме в МИС…». При отсутствии настроенного 1Denta задача пропускается, дубли не создаются.
- **Деактивация и архив автозадач** — невыполненные автозадачи прошлых дней деактивируются ночью (00:05 МСК) и доступны во вкладке «Архив».
- **Бейдж «⚡ Авто»** — автоматически созданные задачи помечаются бейджем; у невыполненной автозадачи без исполнителя отображается «Общая задача».

**Расписание**
- **Переработанная сетка дня** — выше слоты, шире колонки врачей, крупнее карточки приёмов (видны имя пациента, телефон и услуга). Появилась шапка врача с аватаром, именем и специализацией/должностью из справочника ресурсов 1Denta.
- **Закрепление шапки и колонки времени** — при прокрутке строка с врачами остаётся сверху, а колонка времени — слева (sticky по обеим осям).
- **Убран баннер ИИ-ассистента «Расписание»** — страница начинается сразу со статистических карточек.

**Контроль звонков**
- **Объединение дублей звонков** — несколько вебхуков Новофон по одному сеансу (входящий/завершён/пропущен) сводятся в одну строку истории; звонок считается отвеченным только при реально поднятой линии. Дедупликация применяется и к старым записям, и при приёме новых вебхуков.
- **Сверка статусов с Новофон при синхронизации** — статус, длительность, направление и время звонка приводятся к авторитетным данным АТС (disposition). Синхронизация перезаписывает ранее сохранённые строки, исправляя неверно угаданные статусы (например, короткие звонки).

**Карточка пациента**
- **Авто-запуск ИИ-анализа** — «ИИ-Анализ пациента» запускается автоматически при открытии карточки. Результат кэшируется по «отпечатку» данных пациента и переиспользуется, пока данные не изменились; кнопка «Обновить» запрашивает свежий анализ.

**Синхронизация**
- **Ночная полная синхронизация (03:00 МСК)** — задача `sync_full_daily` теперь охватывает не только Справочники, Пациентов и Расписание, но и Контроль звонков (Новофон) и Маркетинг (1Denta). Добавлены отдельные задачи синхронизации звонков и маркетинга.
